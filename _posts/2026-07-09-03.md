### Backtracking（バックトラッキング）とは

バックトラッキングは、解の候補を一つずつ選びながら深さ優先で探索し、条件を満たさないと分かった時点で選択を取り消して（戻って）別の候補を試す手法です。すべての場合を試す全探索を、無駄な分岐を早期に打ち切る「枝刈り（pruning）」によって効率化します。

### 基本の形

1. 現在の状態から選べる候補を一つずつ選ぶ（choose）
2. 選んだ状態で再帰的に探索を進める（explore）
3. 探索が終わったら選択を取り消し、次の候補を試す（un-choose）

```python
def backtrack(path, choices):
    if is_solution(path):
        result.append(path[:])
        return
    for choice in choices:
        if not is_valid(path, choice):
            continue
        path.append(choice)          # choose
        backtrack(path, choices)     # explore
        path.pop()                   # un-choose
```

### 例: 部分集合（Subsets）

```python
def subsets(nums):
    result = []

    def backtrack(start, path):
        result.append(path[:])
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()

    backtrack(0, [])
    return result
```

### 時間計算量

- 最悪の場合は探索木のすべてのノードを訪れるため指数時間（例: 部分集合はO(2^n)）
- 枝刈り（早期にダメだと分かった分岐を切る）によって実際の実行時間は大きく改善されることが多い

### 使いどころ

- 「すべての組み合わせ・順列・部分集合を求める」問題
- N-Queens、数独、迷路探索のような制約充足問題（CSP）
- DFSと組み合わせて使われることが多く、状態を変更 → 再帰 → 元に戻す、というパターンが核心

### 백트래킹이란?

백트래킹(Backtracking)은 해답의 후보를 하나씩 선택하며 깊이 우선으로 탐색하다가, 조건을 만족하지 못한다고 판단되는 순간 선택을 취소하고(되돌아가서) 다른 후보를 시도하는 기법입니다. 모든 경우를 다 시도하는 완전 탐색을, 가망 없는 분기를 조기에 차단하는 "가지치기(pruning)"로 효율화한 방식입니다.

### 기본 형태

1. 현재 상태에서 선택 가능한 후보를 하나 고른다 (choose)
2. 선택한 상태로 재귀적으로 탐색을 진행한다 (explore)
3. 탐색이 끝나면 선택을 취소하고 다음 후보를 시도한다 (un-choose)

```python
def backtrack(path, choices):
    if is_solution(path):
        result.append(path[:])
        return
    for choice in choices:
        if not is_valid(path, choice):
            continue
        path.append(choice)          # choose
        backtrack(path, choices)     # explore
        path.pop()                   # un-choose
```

### 예시: 부분집합(Subsets)

```python
def subsets(nums):
    result = []

    def backtrack(start, path):
        result.append(path[:])
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()

    backtrack(0, [])
    return result
```

### 시간 복잡도

- 최악의 경우 탐색 트리의 모든 노드를 방문하므로 지수 시간이 걸림 (예: 부분집합은 O(2^n))
- 가지치기(조건을 만족하지 못하는 분기를 조기에 차단)를 통해 실제 실행 시간은 크게 개선되는 경우가 많음

### 언제 사용하면 좋은가

- "모든 조합/순열/부분집합을 구하라"는 문제
- N-Queens, 스도쿠, 미로 탐색 같은 제약 충족 문제(CSP)
- DFS와 함께 자주 쓰이며, 상태 변경 → 재귀 → 원상복구라는 패턴이 핵심
