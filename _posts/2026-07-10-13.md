### 問題（Medium）

整数配列 `nums` と整数 `target` が与えられる。インデックスが互いに異なる3つの数を選び、その合計が `target` に最も近くなるようにする。その合計値を返す（解はちょうど1つだけ存在すると仮定してよい）。

```text
入力: nums = [-1,2,1,-4], target = 1
出力: 2  （-1 + 2 + 1 = 2 が最も近い）

入力: nums = [0,0,0], target = 1
出力: 0
```

制約: `3 <= nums.length <= 500`

### アプローチ: ソート + Two Pointers

前回の[3Sum]({% post_url 2026-07-10-12 %})とほぼ同じ骨格で、「合計が0かどうか」ではなく「合計がtargetにどれだけ近いか」を比較する点だけが違う。

1. `nums` をソートし、1つ目の数 `nums[i]` を固定する
2. 残りの区間で `left`/`right` の2つのポインタを両端から動かしながら合計を計算する
3. これまでの最良の答え（`result`）と比べて、今回の合計の方が `target` に近ければ `result` を更新する
4. 合計がちょうど `target` と一致したら、それ以上近い値は存在しないのでその場で返してよい
5. 合計が `target` より小さければ `left` を進めて合計を大きくし、大きければ `right` を戻して合計を小さくする

3Sumと違い、重複を除く処理は不要（"重複しないトリプレット一覧"ではなく"最も近い合計値"だけを1つ求めればよいため）。

### コード

```python
class Solution(object):
    def threeSumClosest(self, nums, target):
        """
        :type nums: List[int]
        :type target: int
        :rtype: int
        """
        nums.sort()
        n = len(nums)
        result = nums[0] + nums[1] + nums[2]
        for i in range(n - 2):
            left, right = i + 1, n - 1
            while left < right:
                current = nums[i] + nums[left] + nums[right]
                if abs(target - current) < abs(target - result):
                    result = current
                if current == target:
                    return result
                elif current < target:
                    left += 1
                else:
                    right -= 1
        return result
```

### 時間・空間計算量

- ソートに `O(n log n)`、外側のループ `i` が `O(n)`、内側のTwo Pointersが最悪 `O(n)` なので、全体は `O(n^2)`
- 追加の空間は定数個の変数のみで `O(1)`

### 문제 (Medium)

정수 배열 `nums`와 정수 `target`이 주어진다. 서로 다른 인덱스 3개를 골라 그 합이 `target`에 가장 가깝게 만든다. 그 합을 반환한다 (정답은 정확히 하나만 존재한다고 가정해도 된다).

```text
입력: nums = [-1,2,1,-4], target = 1
출력: 2  (-1 + 2 + 1 = 2 가 가장 가까움)

입력: nums = [0,0,0], target = 1
출력: 0
```

제약: `3 <= nums.length <= 500`

### 접근 방법: 정렬 + 투 포인터

앞서 다룬 [3Sum]({% post_url 2026-07-10-12 %})과 골격이 거의 같고, "합이 0인지"가 아니라 "합이 target에 얼마나 가까운지"를 비교한다는 점만 다르다.

1. `nums`를 정렬하고, 첫 번째 수 `nums[i]`를 고정한다
2. 나머지 구간에서 `left`/`right` 두 포인터를 양 끝에서 움직이며 합을 계산한다
3. 지금까지의 최선의 답(`result`)과 비교해서, 이번 합이 `target`에 더 가까우면 `result`를 갱신한다
4. 합이 정확히 `target`과 같아지면 그보다 더 가까운 값은 없으므로 그 자리에서 바로 반환해도 된다
5. 합이 `target`보다 작으면 `left`를 옮겨 합을 키우고, 크면 `right`를 옮겨 합을 줄인다

3Sum과 달리 중복 제거 처리는 필요 없다("중복 없는 트리플렛 목록"이 아니라 "가장 가까운 합" 하나만 구하면 되기 때문).

### 코드

```python
class Solution(object):
    def threeSumClosest(self, nums, target):
        """
        :type nums: List[int]
        :type target: int
        :rtype: int
        """
        nums.sort()
        n = len(nums)
        result = nums[0] + nums[1] + nums[2]
        for i in range(n - 2):
            left, right = i + 1, n - 1
            while left < right:
                current = nums[i] + nums[left] + nums[right]
                if abs(target - current) < abs(target - result):
                    result = current
                if current == target:
                    return result
                elif current < target:
                    left += 1
                else:
                    right -= 1
        return result
```

### 시간·공간 복잡도

- 정렬에 `O(n log n)`, 바깥 루프 `i`가 `O(n)`, 안쪽 투 포인터 탐색이 최악의 경우 `O(n)`이므로 전체는 `O(n^2)`
- 추가 공간은 상수 개의 변수뿐이라 `O(1)`
