### 問題（Medium）

文字列の配列 `strs` が与えられる。アナグラム同士（同じ文字を並び替えて作れる文字列同士）をグループ化して返す。順序は自由。

```text
入力: strs = ["eat","tea","tan","ate","nat","bat"]
出力: [["bat"],["nat","tan"],["ate","eat","tea"]]

入力: strs = [""]
出力: [[""]]

入力: strs = ["a"]
出力: [["a"]]
```

制約: `1 <= strs.length <= 10^4`、各文字列は小文字の英字のみ

### アプローチ: ソート済み文字列をキーにしたハッシュマップ

アナグラムかどうかを判定する一番簡単な方法は、文字を並び替えて（ソートして）比較すること。"eat" も "tea" も "ate" も、文字を並び替えると同じ `"aet"` になる。つまり「ソート後の文字列」をハッシュマップのキーにすれば、同じアナグラムのグループは自動的に同じキーの下に集まる。

1. 空のハッシュマップ `groups`（キー: ソート済み文字列 → 値: 元の文字列のリスト）を用意する
2. `strs` の各文字列 `s` について、`key = ''.join(sorted(s))` を計算する
3. `key` がまだ `groups` に無ければ、空リストを作ってから `s` を追加する。すでにあれば、そのリストに `s` を追加するだけ
4. 最後に `groups` の値（リストたち）だけを取り出して返す

このロジック自体は2つの書き方で実装できる。

#### 採用した解法: 通常のdict

```python
groups = {}

for s in strs:
    key = ''.join(sorted(s))
    if key not in groups:
        groups[key] = []
    groups[key].append(s)

return list(groups.values())
```

`key` が初めて出てきたときに、`if key not in groups:` で明示的に空リストを作ってから追加している。

#### 比較用のもう一つの解法: defaultdict

```python
from collections import defaultdict

groups = defaultdict(list)
for s in strs:
    key = ''.join(sorted(s))
    groups[key].append(s)
return list(groups.values())
```

`defaultdict(list)` を使うと、存在しないキーに初めてアクセスしたとき自動的に空リストが作られるため、「キーがあるか確認してから初期化する」処理が不要になり、`if key not in groups: ...` の3行がまるごと消える。

### 比較

| 実装 | キーの初期化 | コード量 | 追加の import |
|---|---|---|---|
| 通常のdict（採用） | `if key not in groups: groups[key] = []` を明示的に書く | やや多い | 不要 |
| defaultdict | アクセス時に自動で空リストが作られる | 少ない | `collections.defaultdict` が必要 |

どちらもアルゴリズム自体（ソート済み文字列をキーにする）は完全に同じで、時間・空間計算量も変わらない。違いは「キーの初期化を自分で書くか、`defaultdict` に任せるか」というスタイルの差だけ。

### 時間・空間計算量

- 文字列の数を `n`、各文字列の最大長を `k` とすると、各文字列のソートに `O(k log k)` かかり、それを `n` 個の文字列すべてに行うので時間計算量は `O(n · k log k)`
- 結果を保持するハッシュマップに `O(n · k)` の空間が必要

### 문제 (Medium)

문자열 배열 `strs`가 주어진다. 애너그램(같은 문자를 재배열해서 만들 수 있는 문자열들)끼리 그룹으로 묶어 반환한다. 순서는 상관없다.

```text
입력: strs = ["eat","tea","tan","ate","nat","bat"]
출력: [["bat"],["nat","tan"],["ate","eat","tea"]]

입력: strs = [""]
출력: [[""]]

입력: strs = ["a"]
출력: [["a"]]
```

제약: `1 <= strs.length <= 10^4`, 각 문자열은 소문자 영문자로만 구성

### 접근 방법: 정렬한 문자열을 키로 쓰는 해시맵

애너그램인지 판단하는 가장 간단한 방법은 문자를 재배열(정렬)해서 비교하는 것이다. "eat"도 "tea"도 "ate"도 문자를 정렬하면 모두 같은 `"aet"`가 된다. 즉 "정렬한 문자열"을 해시맵의 키로 쓰면, 같은 애너그램 그룹이 자동으로 같은 키 아래 모인다.

1. 빈 해시맵 `groups`(키: 정렬된 문자열 → 값: 원래 문자열들의 리스트)를 준비한다
2. `strs`의 각 문자열 `s`에 대해 `key = ''.join(sorted(s))`를 계산한다
3. `key`가 아직 `groups`에 없으면 빈 리스트를 만든 뒤 `s`를 추가한다. 이미 있으면 그 리스트에 `s`를 추가하기만 하면 된다
4. 마지막으로 `groups`의 값들(리스트들)만 꺼내서 반환한다

이 로직 자체는 두 가지 방식으로 구현할 수 있다.

#### 채택한 풀이: 일반 dict

```python
groups = {}

for s in strs:
    key = ''.join(sorted(s))
    if key not in groups:
        groups[key] = []
    groups[key].append(s)

return list(groups.values())
```

`key`가 처음 등장했을 때 `if key not in groups:`로 명시적으로 빈 리스트를 만든 뒤 추가하고 있다.

#### 비교용 다른 풀이: defaultdict

```python
from collections import defaultdict

groups = defaultdict(list)
for s in strs:
    key = ''.join(sorted(s))
    groups[key].append(s)
return list(groups.values())
```

`defaultdict(list)`를 쓰면 존재하지 않는 키에 처음 접근할 때 자동으로 빈 리스트가 만들어지기 때문에, "키가 있는지 확인하고 초기화하는" 처리가 필요 없어져서 `if key not in groups: ...` 세 줄이 통째로 사라진다.

### 비교

| 구현 | 키 초기화 | 코드량 | 추가 import |
|---|---|---|---|
| 일반 dict (채택) | `if key not in groups: groups[key] = []`를 명시적으로 작성 | 다소 많음 | 불필요 |
| defaultdict | 접근 시 자동으로 빈 리스트가 만들어짐 | 적음 | `collections.defaultdict` 필요 |

둘 다 알고리즘 자체(정렬한 문자열을 키로 쓰는 것)는 완전히 동일하고, 시간·공간 복잡도도 같다. 차이는 "키 초기화를 직접 쓰느냐, `defaultdict`에 맡기느냐"라는 스타일 차이뿐이다.

### 시간·공간 복잡도

- 문자열 개수를 `n`, 각 문자열의 최대 길이를 `k`라 하면, 각 문자열을 정렬하는 데 `O(k log k)`가 걸리고 이를 `n`개 문자열 전부에 대해 하므로 시간 복잡도는 `O(n · k log k)`
- 결과를 담는 해시맵에 `O(n · k)`의 공간이 필요하다
