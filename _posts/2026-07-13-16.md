### 問題（Medium）

2つの空でない連結リスト `l1`、`l2` が与えられる。それぞれ0以上の整数を表し、各桁が**逆順**（1の位が先頭）でノードに格納されている。2つの数を足し合わせ、その結果も同じ形式の連結リストとして返す。

```text
入力: l1 = [2,4,3], l2 = [5,6,4]
出力: [7,0,8]
説明: 342 + 465 = 807

入力: l1 = [0], l2 = [0]
出力: [0]

入力: l1 = [9,9,9,9,9,9,9], l2 = [9,9,9,9]
出力: [8,9,9,9,0,0,0,1]
```

制約: 各リストのノード数は `1〜100`、各ノードの値は `0〜9`

### アプローチ: 連結リスト + 繰り上がりのシミュレーション

数字が逆順（下の位から）に並んでいるのがポイントで、これはちょうど筆算で下の桁から足し算していく順番と一致する。なので、2つのリストを先頭から同時に辿りながら、その場で足し算＋繰り上げ処理をしていけばよい。

1. 結果用の連結リストを作るため、番兵（dummy）ノードを1つ用意し、`current` でその末尾を指す
2. `carry`（繰り上がり）を0で初期化する
3. `l1`・`l2`・`carry` のいずれかが残っている間、繰り返す：
   - それぞれのノードの値（存在しなければ0）を取り出す
   - `total = val1 + val2 + carry` を計算する
   - 新しい繰り上がり `carry = total // 10`、今の桁の値 `digit = total % 10` を求める
   - `digit` を新しいノードとして `current.next` に追加し、`current` を1つ進める
   - `l1`・`l2` をそれぞれ次のノードへ進める（存在すれば）
4. ループ終了時点で `carry` も含めてすべて処理し終えているので、`dummy.next` が答えの先頭ノード

`carry` をループの継続条件にも含めているため、`9999 + 1` のように最後の桁で新しく繰り上がりが発生するケース（結果の桁数が増える場合）も自然に処理できる。

### コード

```python
# Definition for singly-linked list.
# class ListNode(object):
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution(object):
    def addTwoNumbers(self, l1, l2):
        """
        :type l1: Optional[ListNode]
        :type l2: Optional[ListNode]
        :rtype: Optional[ListNode]
        """
        dummy = ListNode(0)
        current = dummy
        carry = 0

        while l1 or l2 or carry:
            if l1:
                val1 = l1.val
            else:
                val1 = 0

            if l2:
                val2 = l2.val
            else:
                val2 = 0

            total = val1 + val2 + carry
            carry = total // 10
            digit = total % 10

            current.next = ListNode(digit)
            current = current.next

            if l1:
                l1 = l1.next
            if l2:
                l2 = l2.next

        return dummy.next
```

### 時間・空間計算量

- 長い方のリストの長さを `n` とすると、各ノードを1回ずつ処理するだけなので時間計算量は `O(n)`
- 出力用に新しいリストを作るため、空間計算量も `O(n)`（結果を格納する分を除けば追加の空間は定数個の変数のみ）

### 문제 (Medium)

비어 있지 않은 연결 리스트 `l1`, `l2`가 주어진다. 각각 음이 아닌 정수를 나타내며, 각 자릿수가 **역순**(1의 자리가 맨 앞)으로 노드에 저장되어 있다. 두 수를 더한 결과를 같은 형식의 연결 리스트로 반환한다.

```text
입력: l1 = [2,4,3], l2 = [5,6,4]
출력: [7,0,8]
설명: 342 + 465 = 807

입력: l1 = [0], l2 = [0]
출력: [0]

입력: l1 = [9,9,9,9,9,9,9], l2 = [9,9,9,9]
출력: [8,9,9,9,0,0,0,1]
```

제약: 각 리스트의 노드 수는 `1~100`, 각 노드 값은 `0~9`

### 접근 방법: 연결 리스트 + 자리올림 시뮬레이션

숫자가 역순(낮은 자리부터)으로 저장되어 있다는 게 핵심인데, 이는 손으로 덧셈할 때 낮은 자리부터 더해가는 순서와 정확히 일치한다. 그래서 두 리스트를 앞에서부터 동시에 따라가면서, 그 자리에서 바로 덧셈과 자리올림 처리를 하면 된다.

1. 결과용 연결 리스트를 만들기 위해 더미(dummy) 노드를 하나 준비하고, `current`가 그 끝을 가리키게 한다
2. `carry`(자리올림)를 0으로 초기화한다
3. `l1`, `l2`, `carry` 중 하나라도 남아 있는 동안 반복한다:
   - 각 노드의 값(없으면 0)을 꺼낸다
   - `total = val1 + val2 + carry`를 계산한다
   - 새로운 자리올림 `carry = total // 10`, 이번 자리의 값 `digit = total % 10`을 구한다
   - `digit`을 새 노드로 만들어 `current.next`에 붙이고, `current`를 한 칸 옮긴다
   - `l1`, `l2`를 각각 다음 노드로 진행시킨다 (존재하면)
4. 반복이 끝나는 시점엔 `carry`까지 전부 처리된 상태이므로, `dummy.next`가 답의 첫 노드가 된다

`carry`를 반복 조건에 포함시켰기 때문에, `9999 + 1`처럼 마지막 자리에서 새로 자리올림이 발생해 결과의 자릿수가 늘어나는 경우도 자연스럽게 처리된다.

### 코드

```python
# Definition for singly-linked list.
# class ListNode(object):
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution(object):
    def addTwoNumbers(self, l1, l2):
        """
        :type l1: Optional[ListNode]
        :type l2: Optional[ListNode]
        :rtype: Optional[ListNode]
        """
        dummy = ListNode(0)
        current = dummy
        carry = 0

        while l1 or l2 or carry:
            if l1:
                val1 = l1.val
            else:
                val1 = 0

            if l2:
                val2 = l2.val
            else:
                val2 = 0

            total = val1 + val2 + carry
            carry = total // 10
            digit = total % 10

            current.next = ListNode(digit)
            current = current.next

            if l1:
                l1 = l1.next
            if l2:
                l2 = l2.next

        return dummy.next
```

### 시간·공간 복잡도

- 더 긴 리스트의 길이를 `n`이라 하면, 각 노드를 한 번씩만 처리하므로 시간 복잡도는 `O(n)`
- 출력용 새 리스트를 만들어야 하므로 공간 복잡도도 `O(n)` (결과를 담는 부분을 빼면 추가 공간은 상수 개의 변수뿐)
