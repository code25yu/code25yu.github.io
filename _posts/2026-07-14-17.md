### 問題（Easy）

整数配列 `nums` が与えられる。同じ値が**少なくとも2回**出現するなら `true`、すべての要素が異なるなら `false` を返す。

```text
入力: nums = [1,2,3,1]
出力: true  （1がインデックス0と3に出現）

入力: nums = [1,2,3,4]
出力: false

入力: nums = [1,1,1,3,3,4,3,2,4,2]
出力: true
```

制約: `1 <= nums.length <= 10^5`

この問題は同じロジックを3段階で改善していく過程がわかりやすい題材なので、3つのアプローチを順番に比較する。

### アプローチ1: リストで線形探索

最も素朴な方法は、これまで見た値をリストに貯めておき、新しい値が来るたびにリスト全体を探索して同じ値がないか確認すること。

```python
n = len(nums)
seen = []
for index in range(n):
    if nums[index] in seen:
        return True
    seen.append(nums[index])
return False
```

`nums[index] in seen` の部分がリストに対する線形探索（`O(n)`）になるため、これを配列全体（最大`n`回）繰り返すと全体で `O(n^2)` になる。

### アプローチ2: setで早期終了

`seen` をリストではなく**ハッシュセット**にすると、「すでに見たか」の確認が平均 `O(1)` になる。しかも重複を見つけた瞬間にその場で `True` を返して処理を打ち切れる。

```python
seen = set()
for num in nums:
    if num in seen:
        return True
    seen.add(num)
return False
```

アプローチ1と形はほぼ同じだが、`in` の対象を `list` から `set` に変えるだけで計算量が大きく変わる点がポイント。

### アプローチ3: setの長さを比較（採用した解法）

重複がなければ、集合（set）に変換しても要素数は変わらない。逆に重複があれば、集合にした時点で要素数が減る。この性質を使うと、ループを書かずに1行で判定できる。

```python
return len(set(nums)) != len(nums)
```

ただし、アプローチ2と違って**必ず配列全体を1度セットに変換してから**比較するため、先頭付近に重複があってもそこで打ち切ることはできない。その代わりコードは最も簡潔になる。

### 比較

| アプローチ | 実装 | 時間計算量 | 早期終了 |
|---|---|---|---|
| 1. リスト線形探索 | `seen` はlist、`in`で探索 | `O(n^2)` | あり（重複発見時） |
| 2. set + 早期終了 | `seen` はset、`in`で探索 | `O(n)` | あり（重複発見時） |
| 3. set長さ比較 | `len(set(nums)) != len(nums)` | `O(n)` | なし（常に全体を変換） |

アプローチ1→2は「探索方法をlistからsetに変える」ことでの改善、2→3は「ループを書かない簡潔さ」を優先した書き換えで、その代わり早期終了は失っている。実務的には2と3はどちらも `O(n)` なので、どちらを使うかは「早期終了で少し速くしたいか」「コードの簡潔さを取るか」の好みの問題になる。

### 時間・空間計算量（採用した解法）

- `set(nums)` の構築に `O(n)`、`len()` はどちらも `O(1)` なので、全体の時間計算量は `O(n)`
- 集合を保持するために `O(n)` の追加空間が必要

### 문제 (Easy)

정수 배열 `nums`가 주어진다. 같은 값이 **적어도 두 번** 나타나면 `true`, 모든 원소가 서로 다르면 `false`를 반환한다.

```text
입력: nums = [1,2,3,1]
출력: true  (1이 인덱스 0과 3에 등장)

입력: nums = [1,2,3,4]
출력: false

입력: nums = [1,1,1,3,3,4,3,2,4,2]
출력: true
```

제약: `1 <= nums.length <= 10^5`

이 문제는 같은 로직을 세 단계로 개선해가는 과정을 보여주기 좋은 예시라, 세 가지 접근법을 순서대로 비교한다.

### 접근 방법 1: 리스트로 선형 탐색

가장 단순한 방법은, 지금까지 본 값들을 리스트에 모아두고 새 값이 올 때마다 리스트 전체를 뒤져서 같은 값이 있는지 확인하는 것이다.

```python
n = len(nums)
seen = []
for index in range(n):
    if nums[index] in seen:
        return True
    seen.append(nums[index])
return False
```

`nums[index] in seen` 부분이 리스트에 대한 선형 탐색(`O(n)`)이기 때문에, 이를 배열 전체(최대 `n`번)만큼 반복하면 전체는 `O(n^2)`이 된다.

### 접근 방법 2: set으로 조기 종료

`seen`을 리스트 대신 **해시셋**으로 바꾸면, "이미 본 적 있는지" 확인이 평균 `O(1)`이 된다. 게다가 중복을 발견하는 순간 바로 `True`를 반환하고 처리를 끝낼 수 있다.

```python
seen = set()
for num in nums:
    if num in seen:
        return True
    seen.add(num)
return False
```

접근 방법 1과 형태는 거의 같지만, `in`의 대상을 `list`에서 `set`으로 바꾼 것만으로 시간 복잡도가 크게 달라진다는 점이 포인트다.

### 접근 방법 3: set 길이 비교 (채택된 풀이)

중복이 없다면, 집합(set)으로 변환해도 원소 개수는 그대로다. 반대로 중복이 있다면 집합으로 만드는 순간 원소 개수가 줄어든다. 이 성질을 이용하면 반복문 없이 한 줄로 판정할 수 있다.

```python
return len(set(nums)) != len(nums)
```

다만 접근 방법 2와 달리 **반드시 배열 전체를 한 번 set으로 변환한 뒤** 비교하기 때문에, 앞쪽에 중복이 있어도 그 지점에서 바로 끝낼 수는 없다. 대신 코드는 가장 간결해진다.

### 비교

| 접근 방법 | 구현 | 시간 복잡도 | 조기 종료 |
|---|---|---|---|
| 1. 리스트 선형 탐색 | `seen`은 list, `in`으로 탐색 | `O(n^2)` | 있음 (중복 발견 시) |
| 2. set + 조기 종료 | `seen`은 set, `in`으로 탐색 | `O(n)` | 있음 (중복 발견 시) |
| 3. set 길이 비교 | `len(set(nums)) != len(nums)` | `O(n)` | 없음 (항상 전체를 변환) |

접근 방법 1→2는 "탐색 방식을 list에서 set으로 바꾸는" 개선이고, 2→3은 "반복문을 안 쓰는 간결함"을 우선한 방식으로, 대신 조기 종료를 포기한 것이다. 실무에서는 2와 3 모두 `O(n)`이므로, 어느 쪽을 쓸지는 "조기 종료로 조금이라도 더 빠르게 할지" vs "코드의 간결함을 취할지"의 취향 문제에 가깝다.

### 시간·공간 복잡도 (채택된 풀이)

- `set(nums)`를 만드는 데 `O(n)`, `len()`은 둘 다 `O(1)`이므로 전체 시간 복잡도는 `O(n)`
- 집합을 보관하기 위해 `O(n)`의 추가 공간이 필요하다
