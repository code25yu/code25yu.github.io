### 問題（Medium）

整数配列 `nums` が与えられる。`nums[i] + nums[j] + nums[k] == 0` となる、インデックスが互いに異なる3つの組（トリプレット）をすべて返す。重複するトリプレットは含めない。

```text
入力: nums = [-1,0,1,2,-1,-4]
出力: [[-1,-1,2],[-1,0,1]]

入力: nums = [0,1,1]
出力: []

入力: nums = [0,0,0]
出力: [[0,0,0]]
```

制約: `3 <= nums.length <= 3000`

### アプローチ: ソート + 二分探索（Two Pointers）

以前紹介した[Two Pointers]({% post_url 2026-07-09-03 %})の考え方を、1つ多い次元に応用したものと考えられる。

1. まず `nums` を昇順にソートする
2. 1つ目の数 `nums[i]` を固定し、残りの区間 `[i+1, n-1]` の中で「合計が `-nums[i]` になる2数」を、ソート済み配列に対する二分探索（左右のポインタを両端から狭める）で探す
3. 左右のポインタの合計が0より小さければ `left` を進めて合計を大きく、0より大きければ `right` を戻して合計を小さくする
4. 合計がちょうど0ならその組を記録し、`left`/`right` を1つずつ動かして次を探す

ソート済みであることが重要で、これにより「答えが見つかったら重複を飛ばす」処理も簡単になる。

#### 重複を除く工夫

- `nums[i]` が直前と同じ値なら、その `i` は既に試した組み合わせと重複するのでスキップする
- 答えを1つ見つけたあと、`left`/`right` が隣と同じ値ならその分だけ進めて重複するトリプレットを避ける

#### 枝刈り

配列がソート済みなので、`nums[i] > 0` になった時点でそれ以降の3数の和は必ず正になり、0にはなり得ない。そこで即座にループを打ち切ることができる。

### コード

```python
class Solution(object):
    def threeSum(self, nums):
        """
        :type nums: List[int]
        :rtype: List[List[int]]
        """
        nums.sort()
        n = len(nums)
        result = []

        for i in range(n - 2):
            if i > 0 and nums[i] == nums[i - 1]:
                continue
            if nums[i] > 0:
                break
            left, right = i + 1, n - 1
            while left < right:
                total = nums[i] + nums[left] + nums[right]
                if total < 0:
                    left += 1
                elif total > 0:
                    right -= 1
                else:
                    result.append([nums[i], nums[left], nums[right]])
                    while left < right and nums[left] == nums[left + 1]:
                        left += 1
                    while left < right and nums[right] == nums[right - 1]:
                        right -= 1
                    left += 1
                    right -= 1

        return result
```

### 時間・空間計算量

- ソートに `O(n log n)`、外側のループ `i` が `O(n)`、内側の二分探索が最悪 `O(n)` なので、全体は `O(n^2)`
- 追加の空間は定数個の変数のみで `O(1)`（結果を格納する配列自体は除く）

### 문제 (Medium)

정수 배열 `nums`가 주어진다. `nums[i] + nums[j] + nums[k] == 0`이 되는, 서로 다른 인덱스 3개로 이루어진 조합(트리플렛)을 모두 반환한다. 중복되는 트리플렛은 포함하지 않는다.

```text
입력: nums = [-1,0,1,2,-1,-4]
출력: [[-1,-1,2],[-1,0,1]]

입력: nums = [0,1,1]
출력: []

입력: nums = [0,0,0]
출력: [[0,0,0]]
```

제약: `3 <= nums.length <= 3000`

### 접근 방법: 정렬 + 투 포인터

이전에 다뤘던 [투 포인터]({% post_url 2026-07-09-03 %})의 개념을 한 차원 더 확장한 형태로 볼 수 있다.

1. 먼저 `nums`를 오름차순으로 정렬한다
2. 첫 번째 수 `nums[i]`를 고정하고, 나머지 구간 `[i+1, n-1]`에서 "합이 `-nums[i]`가 되는 두 수"를 정렬된 배열에 대한 투 포인터(양 끝에서 좁혀가는 방식)로 찾는다
3. 두 포인터의 합이 0보다 작으면 `left`를 옮겨 합을 키우고, 0보다 크면 `right`를 옮겨 합을 줄인다
4. 합이 정확히 0이면 그 조합을 기록하고, `left`/`right`를 하나씩 움직여 다음을 찾는다

정렬되어 있다는 점이 핵심인데, 덕분에 "답을 찾았을 때 중복을 건너뛰는" 처리도 쉬워진다.

#### 중복 제거

- `nums[i]`가 바로 이전 값과 같으면, 그 `i`는 이미 시도한 조합과 겹치므로 건너뛴다
- 답을 하나 찾은 뒤, `left`/`right`가 바로 옆과 같은 값이면 그만큼 더 이동시켜 중복 트리플렛을 피한다

#### 가지치기

배열이 정렬되어 있으므로 `nums[i] > 0`이 되는 순간부터는 이후 세 수의 합이 항상 양수가 되어 0이 될 수 없다. 따라서 그 즉시 반복문을 종료할 수 있다.

### 코드

```python
class Solution(object):
    def threeSum(self, nums):
        """
        :type nums: List[int]
        :rtype: List[List[int]]
        """
        nums.sort()
        n = len(nums)
        result = []

        for i in range(n - 2):
            if i > 0 and nums[i] == nums[i - 1]:
                continue
            if nums[i] > 0:
                break
            left, right = i + 1, n - 1
            while left < right:
                total = nums[i] + nums[left] + nums[right]
                if total < 0:
                    left += 1
                elif total > 0:
                    right -= 1
                else:
                    result.append([nums[i], nums[left], nums[right]])
                    while left < right and nums[left] == nums[left + 1]:
                        left += 1
                    while left < right and nums[right] == nums[right - 1]:
                        right -= 1
                    left += 1
                    right -= 1

        return result
```

### 시간·공간 복잡도

- 정렬에 `O(n log n)`, 바깥 루프 `i`가 `O(n)`, 안쪽 투 포인터 탐색이 최악의 경우 `O(n)`이므로 전체는 `O(n^2)`
- 추가 공간은 상수 개의 변수뿐이라 `O(1)` (결과를 담는 배열 자체는 제외)
