### 問題（Medium）

`2`〜`9`の数字だけからなる文字列 `digits` が与えられる。昔の電話のキーパッドのように、各数字が表せる文字の組み合わせをすべて返す（順序は自由）。

```text
入力: digits = "23"
出力: ["ad","ae","af","bd","be","bf","cd","ce","cf"]

入力: digits = "2"
出力: ["a","b","c"]
```

制約: `digits` の長さは1〜4、各文字は `'2'`〜`'9'`。

### アプローチ: バックトラッキング

数字から文字への対応表（電話のキーパッド）をまず用意する。

```python
phone = { '2' : 'abc', '3' : 'def', '4' : 'ghi',
          '5' : 'jkl', '6' : 'mno', '7' : 'pqrs',
          '8' : 'tuv', '9' : 'wxyz' }
```

あとは「今何文字目を決めているか（`index`）」と「これまで選んだ文字（`path`）」を持って再帰すればよい。

1. `index` が `digits` の長さと同じになったら、`path` を1つの文字列としてまとめて結果に追加し、そこで再帰を止める
2. まだ途中なら、現在の数字に対応する文字を1つずつ選んで（choose）`path` に追加し、次の桁へ再帰する（explore）
3. その桁の可能性を全部試し終えたら、選んだ文字を取り消して（un-choose）別の文字を試す

つまり「2」→「a・b・cのどれか」→「3」→「d・e・fのどれか」…というふうに、桁ごとに枝分かれする木を深さ優先で全部たどるイメージ。`path.append` と `path.pop()` がまさに choose / un-choose にあたる。

### コード

```python
class Solution(object):
    def letterCombinations(self, digits):
        """
        :type digits: str
        :rtype: List[str]
        """
        phone = { '2' : 'abc',
                  '3' : 'def',
                  '4' : 'ghi',
                  '5' : 'jkl',
                  '6' : 'mno',
                  '7' : 'pqrs',
                  '8' : 'tuv',
                  '9' : 'wxyz'}
        result = []
        def backtrack(index, path):
            if index == len(digits):
                result.append(''.join(path))
                return

            current_digit = digits[index]
            letters = phone[current_digit]

            for letter in letters:
                path.append(letter)
                backtrack(index + 1, path)
                path.pop()

        backtrack(0, [])
        return result
```

### 時間・空間計算量

- 数字1つあたり最大4文字（7と9）に対応するため、`digits` の長さが `n` のとき組み合わせの総数は最大 `4^n`
- 各組み合わせを文字列に変換する `''.join(path)` に `O(n)` かかるため、時間計算量は `O(4^n · n)`
- 空間計算量は、再帰の深さが `O(n)`、加えて結果を保存する分が `O(4^n · n)`

### 문제 (Medium)

`2`부터 `9`까지의 숫자로만 이루어진 문자열 `digits`가 주어진다. 옛날 전화기 키패드처럼 각 숫자가 나타낼 수 있는 문자 조합을 전부 반환한다 (순서는 상관없음).

```text
입력: digits = "23"
출력: ["ad","ae","af","bd","be","bf","cd","ce","cf"]

입력: digits = "2"
출력: ["a","b","c"]
```

제약: `digits`의 길이는 1~4, 각 문자는 `'2'`~`'9'`.

### 접근 방법: 백트래킹

먼저 숫자 → 문자 대응표(전화 키패드)를 준비한다.

```python
phone = { '2' : 'abc', '3' : 'def', '4' : 'ghi',
          '5' : 'jkl', '6' : 'mno', '7' : 'pqrs',
          '8' : 'tuv', '9' : 'wxyz' }
```

그다음은 "지금 몇 번째 자리를 정하고 있는지(`index`)"와 "지금까지 고른 문자들(`path`)"을 들고 재귀를 돌리면 된다.

1. `index`가 `digits`의 길이와 같아지면, `path`를 하나의 문자열로 합쳐서 결과에 추가하고 재귀를 멈춘다
2. 아직 남았다면, 현재 숫자에 대응하는 문자를 하나씩 골라서(choose) `path`에 추가하고 다음 자리로 재귀한다(explore)
3. 그 자리에서 가능한 문자를 모두 시도했으면, 골랐던 문자를 취소하고(un-choose) 다른 문자를 시도한다

즉 "2" → "a·b·c 중 하나" → "3" → "d·e·f 중 하나" → ... 처럼 자리마다 갈라지는 트리를 깊이 우선으로 전부 훑는 방식이다. `path.append`와 `path.pop()`이 바로 choose / un-choose에 해당한다.

### 코드

```python
class Solution(object):
    def letterCombinations(self, digits):
        """
        :type digits: str
        :rtype: List[str]
        """
        phone = { '2' : 'abc',
                  '3' : 'def',
                  '4' : 'ghi',
                  '5' : 'jkl',
                  '6' : 'mno',
                  '7' : 'pqrs',
                  '8' : 'tuv',
                  '9' : 'wxyz'}
        result = []
        def backtrack(index, path):
            if index == len(digits):
                result.append(''.join(path))
                return

            current_digit = digits[index]
            letters = phone[current_digit]

            for letter in letters:
                path.append(letter)
                backtrack(index + 1, path)
                path.pop()

        backtrack(0, [])
        return result
```

### 시간·공간 복잡도

- 숫자 하나당 최대 4개의 문자(7과 9)에 대응하므로, `digits`의 길이가 `n`일 때 조합의 총 개수는 최대 `4^n`
- 각 조합을 문자열로 합치는 `''.join(path)`에 `O(n)`이 걸리므로 시간 복잡도는 `O(4^n · n)`
- 공간 복잡도는 재귀 깊이 `O(n)`에, 결과를 저장하는 데 드는 `O(4^n · n)`이 추가된다
