### 問題（Easy）

整数配列 `nums` と整数 `target` が与えられる。2つの数の合計が `target` になる、そのインデックスを返す。答えはちょうど1つだけ存在し、同じ要素を2回使うことはできない。

```text
入力: nums = [2,7,11,15], target = 9
出力: [0,1]  （nums[0] + nums[1] == 9）

入力: nums = [3,2,4], target = 6
出力: [1,2]

入力: nums = [3,3], target = 6
出力: [0,1]
```

制約: `2 <= nums.length <= 10^4`

### アプローチ: ハッシュマップ

まず思いつくのは、すべてのペアを総当たりで確認するブルートフォースだが、これは二重ループになり時間計算量が `O(n^2)` になる。

ハッシュマップを使うと、配列を1回走査するだけで `O(n)` に改善できる。ポイントは「今見ている数」ではなく「今見ている数の相棒（`target - num`）が、すでに出てきていないか」をハッシュマップで確認すること。

1. 空のハッシュマップ `num_to_idx`（数値 → インデックス）を用意する
2. 配列を先頭から見ていき、各要素 `num` について `complement = target - num` を計算する
3. `complement` がすでに `num_to_idx` にあれば、その2つのインデックスが答え
4. なければ、今の `num` とそのインデックスを `num_to_idx` に記録して次へ進む

こうすることで、「後から出てくる数」を待たずに、常に「すでに見た数」の中だけを `O(1)` で検索できる。

### コード

```python
class Solution(object):
    def twoSum(self, nums, target):
        """
        :type nums: List[int]
        :type target: int
        :rtype: List[int]
        """
        num_to_idx = {}  # {num: idx}
        for i, num in enumerate(nums):
            complement = target - num
            if complement in num_to_idx:
                return [num_to_idx[complement], i]
            num_to_idx[num] = i
```

### 時間・空間計算量

- 配列を1回だけ走査し、ハッシュマップへの検索・追加はそれぞれ平均 `O(1)` なので、時間計算量は `O(n)`
- ハッシュマップに最大 `n` 個の要素を保持するため、空間計算量も `O(n)`
- ブルートフォース（二重ループで全ペアを確認）の `O(n^2)` から `O(n)` への改善

### 문제 (Easy)

정수 배열 `nums`와 정수 `target`이 주어진다. 두 수의 합이 `target`이 되는 인덱스를 반환한다. 답은 정확히 하나만 존재하며, 같은 원소를 두 번 사용할 수 없다.

```text
입력: nums = [2,7,11,15], target = 9
출력: [0,1]  (nums[0] + nums[1] == 9)

입력: nums = [3,2,4], target = 6
출력: [1,2]

입력: nums = [3,3], target = 6
출력: [0,1]
```

제약: `2 <= nums.length <= 10^4`

### 접근 방법: 해시맵

가장 먼저 떠올리기 쉬운 방법은 모든 쌍을 전부 확인하는 브루트포스인데, 이는 이중 루프가 되어 시간 복잡도가 `O(n^2)`이 된다.

해시맵을 쓰면 배열을 한 번만 훑으면서 `O(n)`으로 개선할 수 있다. 핵심은 "지금 보고 있는 수"가 아니라 "지금 보고 있는 수의 짝(`target - num`)이 이미 나온 적이 있는지"를 해시맵으로 확인하는 것이다.

1. 빈 해시맵 `num_to_idx`(숫자 → 인덱스)를 준비한다
2. 배열을 앞에서부터 보면서, 각 원소 `num`에 대해 `complement = target - num`을 계산한다
3. `complement`가 이미 `num_to_idx`에 있으면, 그 두 인덱스가 답이다
4. 없으면 지금의 `num`과 그 인덱스를 `num_to_idx`에 기록하고 다음으로 넘어간다

이렇게 하면 "나중에 나올 수"를 기다리지 않고도, 항상 "이미 본 수"들 중에서만 `O(1)`로 검색할 수 있다.

### 코드

```python
class Solution(object):
    def twoSum(self, nums, target):
        """
        :type nums: List[int]
        :type target: int
        :rtype: List[int]
        """
        num_to_idx = {}  # {num: idx}
        for i, num in enumerate(nums):
            complement = target - num
            if complement in num_to_idx:
                return [num_to_idx[complement], i]
            num_to_idx[num] = i
```

### 시간·공간 복잡도

- 배열을 한 번만 순회하고, 해시맵 검색·추가가 각각 평균 `O(1)`이므로 시간 복잡도는 `O(n)`
- 해시맵에 최대 `n`개의 원소를 담을 수 있으므로 공간 복잡도도 `O(n)`
- 브루트포스(이중 루프로 모든 쌍 확인)의 `O(n^2)`에서 `O(n)`으로 개선된 것
