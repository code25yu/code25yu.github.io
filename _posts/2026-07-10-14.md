### 問題（Medium）

`n` 個の整数からなる配列 `nums` と整数 `target` が与えられる。`nums[a] + nums[b] + nums[c] + nums[d] == target` となる、インデックス `a, b, c, d` がすべて異なる**重複のない**四つ組（クアドラプレット）をすべて返す。

```text
入力: nums = [1,0,-1,0,-2,2], target = 0
出力: [[-2,-1,1,2],[-2,0,0,2],[-1,0,0,1]]

入力: nums = [2,2,2,2,2], target = 8
出力: [[2,2,2,2]]
```

制約: `1 <= nums.length <= 200`

### アプローチ: ソート + 二重ループ + Two Pointers

[3Sum]({% post_url 2026-07-10-12 %})にもう1段ループを重ねた形と考えるとわかりやすい。3Sumが「1つ固定＋Two Pointers」なら、4Sumは「2つ固定＋Two Pointers」になる。

1. `nums` をソートする
2. 1つ目の数 `nums[i]`、2つ目の数 `nums[j]` を固定する（`i < j`）
3. 残りの区間 `[j+1, n-1]` を `left`/`right` のTwo Pointersで探索し、4数の合計を `target` と比較しながら両端を狭める

#### 枝刈り（pruning）

3Sumの「`nums[i] > 0` なら打ち切り」をより一般化した形で、`i`・`j` それぞれのループで次の2つを判定する。

- **最小合計が既に大きすぎる場合**：ソート済みなので、現在の位置から選べる最小の4数の和（例えば `nums[i]+nums[i+1]+nums[i+2]+nums[i+3]`）がすでに `target` より大きいなら、これ以上 `i` を進めても状況は悪化するだけなので `break`
- **最大合計がまだ小さすぎる場合**：現在の `i` を使って選べる最大の4数の和（例えば `nums[i]+nums[n-3]+nums[n-2]+nums[n-1]`）がまだ `target` に届かないなら、この `i` では絶対に解が出ないので `continue` して次の `i` へ

`j` のループでも同様の判定を、範囲を1つ内側にずらして行う。

#### 重複を除く工夫

`i` が直前と同じ値ならスキップ、`j` も同様（ただし `j > i + 1` の範囲内で）。答えを見つけたあとは3Sumと同じく `left`/`right` の重複値を飛ばす。

### コード

```python
class Solution(object):
    def fourSum(self, nums, target):
        """
        :type nums: List[int]
        :type target: int
        :rtype: List[List[int]]
        """
        nums.sort()
        result = []
        n = len(nums)

        for i in range(n - 3):
            if i > 0 and nums[i] == nums[i - 1]:
                continue
            if nums[i] + nums[i + 1] + nums[i + 2] + nums[i + 3] > target:
                break
            if nums[i] + nums[n - 3] + nums[n - 2] + nums[n - 1] < target:
                continue

            for j in range(i + 1, n - 2):
                if j > i + 1 and nums[j] == nums[j - 1]:
                    continue

                if nums[i] + nums[j] + nums[j + 1] + nums[j + 2] > target:
                    break
                if nums[i] + nums[j] + nums[n - 2] + nums[n - 1] < target:
                    continue

                left, right = j + 1, n - 1

                while left < right:
                    total = nums[i] + nums[j] + nums[left] + nums[right]
                    if total < target:
                        left += 1
                    elif total > target:
                        right -= 1
                    else:
                        result.append([nums[i], nums[j], nums[left], nums[right]])

                        while left < right and nums[left] == nums[left + 1]:
                            left += 1
                        while left < right and nums[right] == nums[right - 1]:
                            right -= 1

                        left += 1
                        right -= 1
        return result
```

### 時間・空間計算量

- ソートに `O(n log n)`、`i`・`j` の二重ループが `O(n^2)`、内側のTwo Pointersが `O(n)` なので、全体は `O(n^3)`
- 追加の空間は定数個の変数のみで `O(1)`（結果を格納する配列自体は除く）

### 문제 (Medium)

`n`개의 정수로 이루어진 배열 `nums`와 정수 `target`이 주어진다. `nums[a] + nums[b] + nums[c] + nums[d] == target`이 되는, 인덱스 `a, b, c, d`가 모두 서로 다른 **중복 없는** 네 개 조합(쿼드러플렛)을 모두 반환한다.

```text
입력: nums = [1,0,-1,0,-2,2], target = 0
출력: [[-2,-1,1,2],[-2,0,0,2],[-1,0,0,1]]

입력: nums = [2,2,2,2,2], target = 8
출력: [[2,2,2,2]]
```

제약: `1 <= nums.length <= 200`

### 접근 방법: 정렬 + 이중 루프 + 투 포인터

[3Sum]({% post_url 2026-07-10-12 %})에 루프를 하나 더 얹은 형태로 보면 이해하기 쉽다. 3Sum이 "하나 고정 + 투 포인터"라면, 4Sum은 "두 개 고정 + 투 포인터"가 된다.

1. `nums`를 정렬한다
2. 첫 번째 수 `nums[i]`, 두 번째 수 `nums[j]`를 고정한다 (`i < j`)
3. 나머지 구간 `[j+1, n-1]`을 `left`/`right` 투 포인터로 탐색하며, 네 수의 합을 `target`과 비교해가며 양 끝을 좁힌다

#### 가지치기 (pruning)

3Sum의 "`nums[i] > 0`이면 종료"를 좀 더 일반화한 형태로, `i`·`j` 루프 각각에서 다음 두 가지를 판단한다.

- **최소 합이 이미 너무 큰 경우**: 정렬되어 있으므로, 현재 위치에서 고를 수 있는 최소 네 수의 합(예: `nums[i]+nums[i+1]+nums[i+2]+nums[i+3]`)이 이미 `target`보다 크다면, `i`를 더 진행해봐야 상황이 나빠지기만 하므로 `break`
- **최대 합이 아직 부족한 경우**: 현재 `i`로 고를 수 있는 최대 네 수의 합(예: `nums[i]+nums[n-3]+nums[n-2]+nums[n-1]`)이 아직 `target`에 못 미친다면, 이 `i`로는 절대 답이 나올 수 없으므로 `continue`해서 다음 `i`로 넘어간다

`j` 루프에서도 범위를 한 칸 안쪽으로 옮겨 같은 판단을 적용한다.

#### 중복 제거

`i`가 바로 이전 값과 같으면 스킵하고, `j`도 마찬가지로 처리한다 (단 `j > i + 1` 범위 안에서). 답을 찾은 뒤에는 3Sum과 동일하게 `left`/`right`의 중복 값을 건너뛴다.

### 코드

```python

```

### 시간·공간 복잡도

- 정렬에 `O(n log n)`, `i`·`j` 이중 루프가 `O(n^2)`, 안쪽 투 포인터가 `O(n)`이므로 전체는 `O(n^3)`
- 추가 공간은 상수 개의 변수뿐이라 `O(1)` (결과를 담는 배열 자체는 제외)
